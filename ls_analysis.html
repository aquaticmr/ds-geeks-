Time Complexity:

Best Case (O(1)): The best-case scenario occurs when the target element x is found at the first position in the array. This requires only one comparison.

Worst Case (O(n)): The worst-case scenario happens when the target element is either at the last position or not present in the array. In this case, the algorithm needs to check all n elements.

Average Case (O(n)): On average, if the element is present in the array, it will be found after searching through half the list (i.e., n/2 comparisons). However, in Big-O notation, the average case complexity is still linear, i.e., O(n).

Space Complexity:

The space complexity of linear search is O(1) because it only requires a constant amount of extra memory, regardless of the size of the input.
